# Problem Frame Authoring Playbook

## Goal

Turn requirement/spec truth into a first validator-ready problem-frame draft for one use case.

## Workflow

1. Pick one use case.
2. Prefer `CBF` unless the use case is clearly a workpiece-oriented `SWF`.
3. Build the extraction sheet.
4. Draft the required problem-frame files.
5. Mark inferred items.
6. End with open questions and the next handoff.

## External-System Rules

Always clarify:

- authority boundary
- timeout meaning
- retry and duplication behavior
- asynchronous confirmation paths
- audit and reconciliation identifiers
